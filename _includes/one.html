Challenges

  1. Instagram has more number of hashtags and considerably longer post length than twitter and Weibo.
  2. User based scraping is very slow because of the limit for GQL queries. For users > 500 posts we encountered many blocking delays as well.
  3. Data scarcity on Instagram: For hashtag based scraping, we get posts with minimal text which does not help provide any context. Image, post and hashtag have nearly no correlation. The hashtag could be related to either the post content or the picture (for example, a picture of the Empire State Building would have the hashtags #nyc #amaze #empirestatebuilding and the post would say something like “had an amazing day!”). Comments are more motivational and supportive (eg. nice pic, love it, etc) whereas twitter comments are more relevant to the post.
    In order to deal with this, we decided to train our model on data scraped from popular, public users who generally post with longer text and few decent hashtags from which we can derive better context.
  4. For incorporating likes into the model, we need a dataset that captures the distinction/popularity between two different hashtags given that the context of the post is the same. For example,
    Post1 : This year’s theme for Felicity is Superheroes #FestofIIITH
    Post2 : This year’s theme for Felicity is Superheroes #FunatIIITH
    As per our knowledge, there is no such available data source to scrape from that would capture this phenomenon and hence, ranking based on this idea is not possible.
