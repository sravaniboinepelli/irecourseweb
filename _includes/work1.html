The Model - What set us aside from the SOTA:

Preprocessing:

  1. Removed noisy data like emoticons and non-ASCII data and single character hashtags
  2. Replaced numbers, URLs, tagged users etc with corresponding tags (DIGIT, MENTION, URL etc).
  3. Segregate by tag, post and comments for overall context
  4. Handling multiple hashtags: post 1 h1 con1, post 1 h2 con2 etc etc (each hashtag should correspond to the same post)
  5. For training instances tagged with multiple gold-standard hashtags, we copy the instances multiple times, each with one gold standard hashtag following.
  6. Splitting hashtag for better context and accuracy using the Viterbi algorithm

Incorporating likes:

  1. The hashtags are sorted by topics(through clustering/ manually/topic modelling) and ranked by number of likes.
  2. Popularity Ranking Algorithm takes two inputs: The output hashtags from the above model along with the list of hashtags per topic sorted by likes.
  3. Since our model predicts unseen hashtags we can only compare topic based hashtags from the training dataset and newly generated hashtags using a similarity metric. We chose the standard cosine_similarity as our metric. More priority is given to the context than to the level of popularity.
